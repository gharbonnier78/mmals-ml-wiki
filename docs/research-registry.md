# Research registry governance

Diderot maintains four connected registries:

- `data/research-events.json`: dated events and their consequences;
- `data/branches.json`: branch lineage, maturity, qualified claim, and next step;
- `data/repositories.json`: curated GitHub and planned repository mapping;
- `data/publications.json`: GitHub, arXiv, HAL, DOI, and venue states kept separately.

## Conversation-derived material

Raw chats are not published. A conversation may generate a curated event only after the event is normalized, checked for private context, assigned a provenance class, and validated against an artifact when possible.

## Refresh policy

GitHub existence and metadata can be refreshed automatically. Scientific maturity and publication status require review and must not be inferred from repository activity alone.
