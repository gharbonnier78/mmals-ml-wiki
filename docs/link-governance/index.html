Repository handbook

Link governance

Permanent semantic URLs, automated checks, and scheduled refresh.

Permanent semantic URLs

  • Published articles use /go/<slug>.
  • A published slug is never deleted.
  • Renamed concepts redirect to a successor and retain historical explanation.
  • Physical page locations can change without breaking articles.

Automated checks

Internal links

Checked on every push and pull request. A broken internal link fails CI and blocks deployment.

External links

Refreshed weekly. Failures are recorded as warnings until reviewed because third-party services can be temporary.

Preferred reference order

DOI → official publisher or institutional repository → arXiv/HAL → project metadata → lawful archive.
