Continual learning

Stability–plasticity

The tension between preserving useful old functions and adapting enough to learn new ones.

Central evaluated trade-offreviewed 2026-06-18stable /go/ link

In plain language

Too little memory causes forgetting. Too much memory prevents learning. A continual learner must live between those extremes.

Engineering view

Tune memory coefficients with validation-selected policies, report old-task retention and new-task learning separately, and inspect weak seeds. Stability mechanisms include replay, distillation, consolidation, modularity, and route-function preservation.

Research view

The trade-off is not one-dimensional: context competence, route competence, calibration, compute cost, and action safety can shift the apparent optimum.

Formal anchor

\min \; \mathcal L_{new}+\lambda\mathcal L_{memory}

Current evidence

Repeatedly observed across MMALS versions, including negative results where stronger constraints stabilized internal geometry but harmed plasticity.

Boundary

A single final-average-accuracy score can hide severe task imbalance or poor actionability.
