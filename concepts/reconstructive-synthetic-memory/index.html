Memory

Reconstructive and synthetic memory

Two complementary modes: replayable route-function history for audit, and compressed reusable abstractions for efficient operation.

Design programreviewed 2026-06-18stable /go/ link

In plain language

One memory keeps the full story so we can retrace it. The other keeps a compact lesson so the system can act quickly.

Engineering view

Reconstructive audit memory stores enough evidence to replay or reconstruct route-function paths, while synthetic memory compiles reusable representations, summaries, prototypes, or policies. They have different fidelity, cost, and governance requirements.

Research view

The key design problem is controlled compilation: determining what can be compressed without losing causal traceability, scientific reproducibility, or safety-relevant context.

Formal anchor

M = M_{audit} \oplus M_{compiled}
Design program
Current evidence

Conceptually integrated into the MMALS roadmap; implementation remains partial across replay, snapshots, prototypes, and evidence bundles.

Boundary

A compressed memory can hide failure modes; a reconstructive memory can be too expensive for routine inference.
