Memory

Functional memory

Memory that preserves a reusable computation or behavior rather than only parameters, topology, or route identity.

Multi-seed evidencereviewed 2026-06-18stable /go/ link

In plain language

The system tries to keep what an old pathway did, not only where the pathway was.

Engineering view

Functional memory may combine latent distillation, output distillation, route constraints, transform constraints, replay, and snapshots. It should expose both reconstruction fidelity and operational cost.

Research view

Evidence supports functional memory as more meaningful than route-vector memory in controlled continual-learning settings. The remaining question is when each memory component adds causal value beyond generic replay or distillation.

Formal anchor

D_z=\mathbb E\lVert z_{f,current}-z_{f,teacher}\rVert_2^2

Current evidence

Strong internal evidence exists in the v0.7–v0.9 line and later RC2O retention work; benchmark breadth and hardened baselines remain important.

Boundary

Functional memory is not automatically compact, cheap, or beneficial under all context shifts.
