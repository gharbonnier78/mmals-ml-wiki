Inferred context

A distribution q(c|x) over possible situations inferred from the input instead of supplied as a task identifier.

Multi-seed evidencereviewed 2026-06-18stable /go/ link

In plain language

The system is not told which task it is solving. It first estimates what situation the input belongs to, then chooses routes accordingly.

Engineering view

The context posterior can be prototype-based, learned, calibrated, blended, or guarded by a global head. Uncertainty in q(c|x) should propagate into route uncertainty and action gating rather than being discarded.

Research view

Moving from explicit task IDs to inferred context is a central MMALS transition. Evaluation must separate context competence, routing competence, output competence, calibration, and action safety.

Formal anchor

q(c\mid x),\qquad r_i(x)=\sum_c q(c\mid x)r_i(x,c)
Multi-seed evidence
Current evidence

The RC1/RC1b line provided strong controlled evidence on overlapping FashionMNIST contexts, while broader datasets remain under qualification.

Boundary

A good context posterior on one benchmark does not establish general task-free continual learning.
