Routing

Route

The context- and input-dependent allocation of host contributions used to construct a mediated computation.

Implemented primitivereviewed 2026-06-18stable /go/ link

In plain language

A route says how much the system uses each host for the current case.

Engineering view

Routes lie on a probability simplex. Entropy measures diffuseness; dominance measures concentration; cost-aware routing can include latency, memory, energy, or risk.

Research view

Route stability is not functional stability. A route can remain numerically similar while host representations or transforms drift.

Formal anchor

r_i(x,c)=\frac{e^{a_i(x,c)/\tau}}{\sum_j e^{a_j(x,c)/\tau}}
Implemented primitive
Current evidence

Routing is present throughout MMALS, but route-only memory was repeatedly shown to be insufficient as a complete memory object.

Boundary

Sparse or stable routes do not by themselves prove accuracy, memory, or specialization.
