Host

A specialist computational module whose value is defined by its contribution to the wider MMALS system, not only by isolated accuracy.

Implemented primitivereviewed 2026-06-18stable /go/ link

In plain language

A host is one specialist in the ecosystem. It may see the same input as other hosts but learn a different representation.

Engineering view

Hosts can be heterogeneous encoders, experts, modalities, or reusable functional modules. A host should expose its latent signal, cost, route usage, contribution gain, ablation effect, and drift diagnostics.

Research view

Host specialization should be established through multiple converging measurements: route entropy and dominance, contribution gain, ablation impact, representation separation, route-function stability, and cross-seed reproducibility.

Current evidence

Host banks and routing have been implemented since early MMALS prototypes; strong ecological specialization remains mixed and benchmark-dependent.

Boundary

High route weight alone does not prove functional specialization.
