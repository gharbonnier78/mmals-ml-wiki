TPUT

The MMALS system-level layer for turning goals, evidence, cost, memory, and routing into governed decisions.

System design programreviewed 2026-06-18stable /go/ link

In plain language

TPUT asks the whole ecosystem what action is worth taking, not merely which model produced the highest score.

Engineering view

The layer can combine objective state, context posterior, route evidence, memory confidence, compute/latency cost, selector regret, safety gates, and forward/backward control. Keep the operational loop minimal and auditable.

Research view

TPUT is the place to unify geometry, information, dynamics, access, causal integration, and later embodiment without forcing every branch into the core model.

Formal anchor

a^*=\arg\min_a \; Cost(a)+Risk(a)-Utility(a)
System design program
Current evidence

A conceptual and specification layer linked to several MMALS branches; full end-to-end implementation is pending.

Boundary

A system-of-systems layer can become vague unless every signal has a schema, producer, consumer, and decision effect.
